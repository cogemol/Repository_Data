# Repository_Data

Nutzung von MPG.PuRe Daten für verschiedene Anwendungsfälle   

Ein Python Codebook mit Erläuterungen zur Anpassung für individuellen Bedarf. Hier ist immer eine große Abhängigkeit zu den Daten in PuRe gegeben, so dass der Code eher entwickelnd gelistet ist und viele Kommentare enthält, um bessere Anpassungen zu ermöglichen und den Prozess nachvollziehbar zu machen.  

Der Code wurde in z.T. in Anaconda, z.T. in Studio Visual Code erstellt und funktioniert mit Python 3.9., 3.12 und 3.13.  
Er wurde zum Teil unter Zuhilfename von unterschiedlichen AI-Tools erzeugt (überwiegend Chat-AI der GWDG mit Qwen 3 30B A3B Instruct, in geringen Teilen aber auch Gemini über Google Colab, in seiner Funktionalität intellektuell getestet.   
Er befindet sich in einem Beta-Zustand und zeigt Work-in-Progress.

Schritte zur Bearbeitung:  
0. Datenexport aus PuRe für Skripte via Openrefine (pdf)
1. PuRe_Rohdatenextrag (jpynb)
2. Abfrage_Gleichstellungsplan (jpynb)
3. Abfrage_OpenAccess (jpynb)
4. Allgemeine Statistiken (jpynb)

Daten werden im Ordner "Daten" abgelegt und Ergebnisdokumente im Ordner "Results".

Anmerkungen, Verbesserungen und Ideen gerne an cm@mpifg.de
